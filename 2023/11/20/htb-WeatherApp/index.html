/routes/index.js里定义了路由接口，主要有三个：/login，/register，/api/weather。

register只对本地ip开放注册，引用了db.register(username, password)用于注册用户。关于db操作的方法均在database.js中定义

        return new Promise(async (resolve, reject) => {
            try {
                let smt = await this.db.prepare('SELECT username FROM users WHERE username = ? and password = ?');

从上面两段代码可以看到，要获取flag，需要admin身份，在isAdmin中似乎找不到利用点，register中是存在sql注入漏洞的。而要使用register，就要找到SSRF。

对于SSRF，程序提供的方法是get，而注册的方法需要post，这里还需要用到请求走私的知识，node8的http模块很容易通过request splitting（smugging）受到ssrf攻击

https://blog.csdn.net/f_cccc/article/details/116406838

Security Bugs in Practice: SSRF via Request Splitting

这里是它的翻译https://xz.aliyun.com/t/2894

这是一个Node.js的特性，用户发出的http请求路径指定为字符串，但是Node.js最终必须将请求作为原始字节输出。JavaScript支持unicode字符串，因此将字符串转换为字节意味着选择并应用适当的unicode编码。在处理不包含body的请求时，Node.js默认使用latin1，一种单字节编码处理字符（当处理高字节字符时，它将被阶段成其表示的最低字节）

当尝试在路径中使用http控制字符时，通常会被百分号转移

> http.get('http://example.com/\r\n/test').output
[ 'GET /%0D%0A/test HTTP/1.1\r\nHost: example.com\r\nConnection: close\r\n\r\n' ]

然而，由于上述对unicode的处理漏洞可以用于帮助我们进行逃逸。

当node.js8以及以下的版本使用GET请求一个URL时，不会造成逃逸因为这些不是http控制字符

> http.get('http://example.com/\u010D\u010A/test').output
[ 'GET /čĊ/test HTTP/1.1\r\nHost: example.com\r\nConnection: close\r\n\r\n' ]

而当这些字符串被指定为latin1时，这些字符被截断成\r和\n的字节形式。

> Buffer.from('http://example.com/\u{010D}\u{010A}/test', 'latin1').toString()
'http://example.com/\r\n/test'

在Node.js 10之后，这个问题已经被修复，但是Node.js 8和更低版本的服务，如果用到了这个模块，并出现下列情况很可能受到SSRF：

  • 从用户输入接受unicode输入
  • 将这个输入包含到一个外发的http请求路径中
  • 请求的body为空（GET或DELETE）

\r  编码为	 	 		 \u010D
\n	编码为				 \u010A
